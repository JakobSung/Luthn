# 원본 참조

[English](source-references.md)

원본 참조는 원문을 기본적으로 노출하지 않으면서 안전한 지식 기록을 비공개 원본 기록과 연결합니다.

## 저장소 규칙

Luthn 자체에 속하는 공개해도 안전한 제품 구조, 데이터 경계, 구현 코드만 저장합니다. 비공개 원본 문서, 개발 에이전트 협업 메모, 인계 자료, 실행 증거, 로컬 작업 공간 경로, 로컬 조정 파일, 민감한 원문은 커밋하지 않습니다.

## 원본 참조 형태

원본 참조에는 다음 정보를 둡니다.

- `sourceId`: API와 SDK 계약에서 사용하는 표준 공개 원본 식별자
- `sourceType`: `source-event`처럼 공개해도 안전한 원본 분류
- `referenceKind`: `redacted-summary`와 같은 투영 종류
- `redactionState`: `safe-projection-only`와 같은 공개 안전 가림 상태
- 이를 포함하는 Core/위키 투영의 민감도
- 식별자 또는 대상만 담은 감사 메타데이터
- API 계약이 허용할 때만 안전하거나 가려진 요약

보호된 Vault 경계 내부의 기록이 아니라면 비공개 원문을 포함해서는 안 됩니다.

## Core 관계

Core가 관리하는 기록은 식별자로 원본 참조를 가리킬 수 있습니다. 에이전트용 투영에는 가려진 요약이나 안전한 이름만 사용합니다.

## 공개 식별자 이름

새 API, SDK, connector, MCP 계약에서는 `sourceId`를 표준 공개 원본 식별자로 사용합니다. 기존 응답의 `sourceEventId`는 같은 값에 대한 하위 호환 별칭으로 남을 수 있지만, 새 요청 입력은 `sourceId`를 사용해야 합니다.
